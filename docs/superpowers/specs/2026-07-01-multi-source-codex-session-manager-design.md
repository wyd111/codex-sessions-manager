# Multi-Source Codex Session Manager Design

## Goal

Adapt `coramba/codex-sessions-manager` from a single-session-root viewer into a local-only manager that can browse and resume sessions from multiple Codex state homes or session roots, independent of the currently active Codex login mode.

## Context

On this machine, the only complete Codex state home with sessions is `C:\Users\15693\.codex`. It currently uses API key auth and contains both `sessions` and `archived_sessions`. Session JSONL metadata does not include reliable `auth_mode`, account id, or email fields, so old files in one home cannot be safely split into API key vs ChatGPT-account groups after the fact.

Codex login changes can affect which threads the Codex app or picker shows, but this manager should be a direct local file viewer. It should not log in, copy tokens, inspect secrets, or mutate `auth.json`.

## Architecture

The app will support multiple configured session sources. A source can be:

- a full `codexHome`, where the server scans `<codexHome>/sessions` and `<codexHome>/archived_sessions`; or
- a direct `sessionsPath`, for compatibility with the original Docker/WSL workflow.

The Vite middleware will expose a richer `/__sessions_index` response containing `{ source, rel, archive, url }` entries. The browser will fetch file content from safe local URLs generated by the middleware. Parsed sessions will carry `sourceId`, `sourceName`, `codexHome`, `sessionsRoot`, and `archive` metadata.

## Configuration

Preferred new configuration:

```env
CODEX_SESSION_SOURCES=[
  {"id":"default-apikey","name":"Default API Key","codexHome":"C:/Users/15693/.codex"},
  {"id":"chatgpt","name":"ChatGPT Login","codexHome":"C:/Users/15693/.codex-chatgpt"}
]
```

Backward compatibility:

```env
SESSIONS_ROOT_PATH=/home/<user>/.codex/sessions
```

If `CODEX_SESSION_SOURCES` is missing, the app will fall back to `SESSIONS_ROOT_PATH`, and then to the repository-local `sessions` directory.

## User Experience

- Header shows a source filter with `All sources` plus each configured source.
- Grouping adds `By source`.
- Session cards display the source name and whether the file is archived.
- Copy resume command uses the correct source:
  - If source has `codexHome`, copy a PowerShell-friendly command that sets `CODEX_HOME`, changes to `session.cwd`, and runs `codex resume <SESSION_ID>`.
  - If source only has `sessionsPath`, fall back to the original `cd <cwd> && codex resume <SESSION_ID>` command.
- Copy remove command uses the actual file path returned by the server metadata.

## Error Handling

- Missing source directories are returned as source metadata with zero files; the app still loads other sources.
- Malformed `CODEX_SESSION_SOURCES` falls back to single-root behavior and logs a server warning.
- Source ids are sanitized before being used in file URLs.
- File-serving requests are constrained to known source roots to avoid arbitrary path reads.

## Testing

Add Vitest-based tests for the logic that does not require a browser:

- source config parsing and fallback behavior;
- multi-root index generation, including active and archived session roots;
- session parsing preserving source metadata;
- resume/remove command generation for Windows `codexHome` sources and legacy direct session roots.

## Non-Goals

- No login management.
- No mutation of `auth.json` or token storage.
- No attempt to infer historical account identity from old JSONL files without reliable metadata.
- No server-side deletion in this iteration; removal remains a copied command.
